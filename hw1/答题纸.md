# 互联网应用开发技术 - 作业1 答题纸

在本作业的完成过程中，使用人工智能助手进行了界面的代码生成。以下是与 AI 模型（Google DeepMind Assistant）交互时使用的主要提示词（Prompts）：

## 开发过程与提示词记录

### 1. 分析需求与设立工程结构
**提示词 (核心意图)：** 
> *读取作业PDF要求，抽取关键功能点。规划创建 login.html, index.html, details.html, cart.html, order.html, style.css。我们需要使用现代的美观设计，无内联样式，使用语义化的 HTML5 标签。*

**AI 的响应摘要：**
AI提取了PDF内容中的各项要求，建立了一个详尽的“实现计划”(Implementation Plan)。它确认了五个页面的名称，决定使用单一外部CSS文件 `style.css` 来强制样式分离，并采用 Flexbox 和 Grid 布局，定义 CSS Variables 实现更高级的暗色模式体验与主题切换准备。

### 2. 编写全局样式 `style.css` 和登录页 `login.html`
**提示词 (核心意图)：** 
> *首先创建 `style.css`。使用 CSS Variables 配置现代UI的主题色。要求实现全局重置(Reset)、导航栏(Nav)、表单(Forms)与按钮(Buttons)等所有共享组件。*
> *接着编写 `login.html`，使用 `<article>` 或 `<main>` 语义化标签构建登录页的主体卡片，注意表单应当通过外部类选择器控制。*

**AI 的响应摘要：**
AI生成了一个包含700多行的 `style.css`，引入了 Google Fonts `Inter` 字体，使用柔和阴影 (`box-shadow`) 和平滑过渡动画 (`transition`)。接着输出了结构清晰的 `login.html` 表单。

### 3. 构建书籍列表页 `index.html` 与详情页 `details.html`
**提示词 (核心意图)：** 
> *生成两张页面：`index.html` 展示书籍列表，要求使用 `<article>` 和 `<figure>` 标签组织书籍卡片。请包含几本软件工程领域的经典书籍（如重构、人月神话等）。生成 `details.html` 展示《重构》一书的详细信息，包含价格、图文介绍和加入购物车的按钮。*
> *另外，请编写脚本从互联网上（如 Unsplash）拉取一些真实的占位图片，或者使用你自己的工具准备一些本地图片放入 `images/` 目录下，以满足打包提交的要求。*

**AI 的响应摘要：**
AI编写了 Bash 脚本，使用 `curl` 下载了四张符合情境的占位图存放在 `images/` 中。<br>
然后，AI根据规划输出了使用 CSS Grid 布局的 `index.html`，以及页面之间相互跳转完整的导航栏。详情页面也遵循了 HTML5 语义化要求。

### 4. 构建购物车页 `cart.html` 和 订单页 `order.html`
**提示词 (核心意图)：** 
> *生成 `cart.html` 购物车页面，展示已加入购物车的书籍，包含数量增减输入框、移除按钮以及右侧的结算摘要面板。最后生成 `order.html`，展示“支付成功”的动画状态或提示框图标，显示订单编号。*

**AI 的响应摘要：**
AI生成了符合现代电商购物车标准的布局规范。为了页面生动性，还引入了 Feather Icons (基于SVG的轻量级图标库)，用以展现“返回主页”、“减增加数量”等控件逻辑图标，让整个产品在视觉上形成闭环。

## 代码规范与完成度自查
- **[通过]** HTML5语义化标签使用：充分使用了 `<header>, <main>, <article>, <figure>, <section>, <aside>`。
- **[通过]** CSS文件分离管理：页面没有任何 `style="..."` 行内样式干扰（除了 Feather Icon 必要的轻量替代属性）。
- **[通过]** CSS选择器规范：抛弃针对 `<p>`、`<h1>` 的直接滥用，采用了 `.book-card`, `.auth-wrapper`, `.nav-item` 等具名类选择器（Class Selectors）。
- **[通过]** 页面跳转逻辑：5张页面互相链接，构成了完整的前端跳转闭环。
