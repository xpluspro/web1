# 互联网应用开发技术 - 作业1 答题纸 (V2 Tailwind 升级版)

在本作业的完成过程中，使用人工智能助手进行了界面的代码生成与后续的重构。以下是与 AI 模型交互时使用的主要提示词（Prompts）：

## 开发过程与提示词记录

### 1. 初始设计分析与设立工程结构
**提示词 (核心意图)：** 
> *读取作业PDF要求，抽取关键功能点。创造 login.html, index.html, details.html, cart.html, order.html, style.css。我们需要使用现代的美观设计，无内联样式，使用语义化的 HTML5 标签。*

**AI 的响应摘要：**
AI提取了内容，决定使用单一外部CSS文件 `style.css` 实现，并建立详尽的“实现计划”。

### 2. 用户反馈与 Tailwind CSS 重构需求 (V2)
**用户的最新反馈提示词：** 
> *我建议使用tailwind css，同时增加侧边栏（sidebar）支持更多功能（比如图书筛选），同时顶栏应该要有图书搜索框，然后首页应该还要有一个图片滚动栏来做书目推荐的*

**AI 的响应摘要与执行：**
针对这一需求，AI 进行了以下深度的架构和UI重组：
1. **构建工具引入：** AI 建议初始化 `package.json` 并引入 Node.js 环境下的 `tailwindcss` CLI。通过 `tailwind.config.js` 配置字体和主题色，然后通过 `npx tailwindcss -i ./src/input.css -o ./style.css` 命令编译出最终合规的（无内联样式的）外部 CSS 文件。
2. **构建共享布局 (Shared Layout)：** 
   - **顶栏导航 (Navbar):** 新增了一个带搜索图标的全局搜索框。
   - **响应式侧边栏 (Sidebar):** 引入了使用 Flexbox/Grid 与左侧固定的多功能过滤侧边栏（包含分类和价格区间筛选）。
3. **首页特性增强 (Homepage Carousel)：** 在 `index.html` 的核心区域增加了一个基于原生 JavaScript 的定时图片滚动栏 (Carousel)，用于展示"本周推荐"的新书。
4. **全面 Tailwind 迁移：** AI 将所有的5个页面 (`login.html`, `index.html`, `details.html`, `cart.html`, `order.html`) 里的原普通 CSS 类全部推翻，严格使用了 Tailwind 的 Utility Classes，保证了高度一致的设计语言（如 `bg-gray-50`, `text-primary-600`, `shadow-sm`, `rounded-2xl` 等），极大地提升了页面的现代感和模块化。

### 3. 增强用户流与轮播图功能 (V3)
**用户的反馈提示词：**
> *一个逻辑问题，购物车里前往结账之后应该前往订单确认页面而不是直接支付成功，而且你的sidebar功能没做完善，点击具体的类别之后无法跳转，然后图片滚动栏应该要能循环滚动，现在至少动画效果不是循环的*

**AI 的响应摘要与执行：**
针对细节体验的反馈，AI 进行了以下精准修复：
1. **订单流程修正：** 将原先立刻显示“支付成功”的 `order.html` 重命名为 `success.html`，然后重新生成了一个真正的 `order.html` 结算表单页面，包含了收货地址填写和支付方式选择。
2. **侧边栏高亮交互：** 在 `index.html` 的侧边栏过滤选项上绑定了原生的 JavaScript 点击事件，当用户点击特定分类时，动态切换其选中的激活状态 (`bg-primary-50`)，提升了界面可交互的视觉反馈。
3. **首尾相连的无缝轮播图：** 重构了 Carousel 轮播图的 JavaScript 脚本。通过 `cloneNode` 克隆了首尾的幻灯片节点，当动画滚动到边缘界限时，利用 `transitionend` 监听器无缝静默替换节点位置，从而彻底解决了回退时的逆向动画问题，并重置了定时器以防止交互冲突，实现了商业级的顺滑循环。

### 4. 增强购物车交互与订单明细 (V4)
**用户的反馈提示词：**
> *购物车里面无法移出item，同时订单确认页面里面，订单具体有哪些书应该要放在收货人信息上面并且同时展示书名，书的图片和价格的*

**AI 的响应摘要与执行：**
1. **购物车动态移除与重计算：** 在 `cart.html` 中注入了原生 JS 逻辑，给每一个“移除”按钮绑定了点击事件，实现了动态 `row.remove()` 操作，并且会实时重算小计和总价。同时数量的 `+`、`-` 按钮也实现了动态修改和价格刷新。
2. **丰富的订单结算明细：** 深度改造 `order.html`，在原本的收货人表单上方，新增了一个视觉层次分明的“订单商品”详情列表模块 (`ul.divide-y`)。此列表将展示所购书籍的缩略图、书名、作者、单价及选购数量，帮助用户在结算前清晰确认交易细节。
